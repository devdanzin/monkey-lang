## Notes Organization Research

Did deep research across multiple sources:
- OpenClaw docs (memory system, workspace layout, compaction, memory v2 research doc)
- MemGPT/Letta architecture (tiered memory: core/archival/recall)
- PARA method and Zettelkasten (good principles but overkill for single agent)
- Community best practices (MEMORY.md <100 lines, daily logs <4KB, natural language for search)
- Anti-patterns (over-organizing, deep hierarchies, storing everything, no reflection)

Key takeaway: don't over-organize. Vector search handles retrieval. Keep structure shallow (max 2 levels). The Retain/Recall/Reflect cycle is what actually makes memory useful.
